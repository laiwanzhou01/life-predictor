## 🚀 启用 GitHub Pages（必需步骤）

### 方法一：使用 GitHub Actions（推荐）

我已经为您创建了自动部署配置，但您需要在 GitHub 上启用 GitHub Pages：

#### 步骤：

1. **访问仓库设置**
   - 打开 https://github.com/laiwanzhou01/life-predictor
   - 点击顶部的 `Settings` (设置) 标签

2. **配置 Pages**
   - 在左侧菜单中找到 `Pages`
   - 在 `Source` (来源) 下拉菜单中选择：
     - **GitHub Actions** （而不是 Deploy from a branch）
   
3. **等待部署**
   - 保存设置后，GitHub Actions 会自动开始部署
   - 点击仓库顶部的 `Actions` 标签查看部署进度
   - 等待绿色的 ✅ 标记出现

4. **访问网站**
   - 部署成功后，网站将在以下地址可用：
   - **https://laiwanzhou01.github.io/life-predictor/**

## 🐛 常见问题

### Q1: 页面显示 404
**原因**: Pages 可能还未启用或正在部署中  
**解决**: 
- 检查 Settings > Pages 是否已配置
- 查看 Actions 标签，确认部署成功
- 等待 5-10 分钟后再试

### Q2: 样式没有加载
**原因**: 资源路径可能不正确  
**解决**: 
- 本项目使用相对路径，应该没有问题
- 清除浏览器缓存后重试
- 打开浏览器开发者工具(F12)查看错误

### Q3: 部署失败
**原因**: GitHub Actions 权限不足  
**解决**:
1. 访问 Settings > Actions > General
2. 在 "Workflow permissions" 中选择 "Read and write permissions"
3. 勾选 "Allow GitHub Actions to create and approve pull requests"
4. 保存设置并重新触发部署
